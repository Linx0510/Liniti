## 4 Разработка веб-приложения Liniti

### 4.1 Реализация веб-приложения

При реализации сайта в репозитории **Liniti** была выбрана серверная архитектура на основе **Node.js + Express + EJS**. Точка входа приложения настраивает шаблонизатор, подключает статические ресурсы, сессии пользователя и базовые middleware безопасности. После этого подключаются маршруты публичной части, авторизации, API и административной панели (листинг 1).

**Листинг 1 – Инициализация приложения и подключение маршрутов**

```js
app.set('view engine', 'ejs');
app.set('views', path.join(__dirname, 'views'));

app.use(express.urlencoded({ extended: true, limit: '50mb' }));
app.use(express.json({ limit: '50mb' }));
app.use(express.static(path.join(__dirname, 'public')));

app.use(attachCurrentUser);
app.use(ensureCsrfToken);

app.use(pageRoutes);
app.use(authRoutes);
app.use(apiRoutes);
app.use('/admin', adminRoutes);
```

Главная страница реализована как лендинг с презентацией сервиса, секциями преимуществ, сценариев использования и блоком обратной связи. В интерфейсе предусмотрены переходы к регистрации и входу, что упрощает первый пользовательский сценарий для новых участников платформы (рисунок 1).

**Рисунок 1 – Главная страница (лендинг) сайта Liniti**

Ключевая пользовательская часть системы — страница ленты работ (`/lenta`). На ней выводятся карточки проектов, содержащие:
- автора работы;
- галерею изображений проекта;
- название и переход на детальную страницу;
- теги категорий;
- действия пользователя (лайк, жалоба, подписка).

Для каждой карточки применяется серверная подстановка данных из базы, включая массив изображений, список категорий и признак, поставлен ли текущим пользователем лайк (рисунок 2).

**Рисунок 2 – Лента работ с интерактивными карточками проектов**

Для удобства поиска контента реализован виджет фильтрации по категориям в правой панели. Фильтр поддерживает множественный выбор, отображение числа выбранных пунктов и быстрый сброс. Такой подход особенно полезен при большом объёме публикуемых работ (рисунок 3).

**Рисунок 3 – Блок фильтрации работ по категориям**

На странице отдельной работы (`/works/:id`) реализована расширенная карточка проекта с изображениями, описанием, датой публикации, списком категорий и переходом в профиль автора. Галерея переключается локальным JavaScript-кодом без перезагрузки страницы, что повышает удобство просмотра портфолио (рисунок 4).

**Рисунок 4 – Страница отдельной работы с галереей изображений**

Профиль пользователя (`/profile`, `/profile/:id`) отображает публичные сведения об авторе, опубликованные работы, рейтинг и информацию о подписках. Для владельца профиля дополнительно показываются работы в статусе модерации (`pending`), что разделяет публичную и служебную части контента (рисунок 5).

**Рисунок 5 – Профиль пользователя и его портфолио**

Для публикации новых работ реализована защищённая страница создания (`/works/create`) с загрузкой до 10 изображений. Загрузка выполняется через `multer` с ограничением размера файлов и проверкой MIME-типа, что позволяет отклонять недопустимые вложения ещё на этапе приёма запроса (листинг 2).

**Листинг 2 – Ограничения при загрузке изображений работы**

```js
const workUpload = multer({
  storage: workStorage,
  limits: {
    fileSize: 10 * 1024 * 1024,
    files: MAX_WORK_IMAGES,
  },
  fileFilter: (_req, file, cb) => {
    if (file.mimetype && file.mimetype.startsWith('image/')) {
      return cb(null, true);
    }
    return cb(new Error('Разрешены только изображения'));
  },
});
```

Дополнительно в системе реализованы:
- чат между пользователями (`/chat`);
- раздел заказов (`/orders`, `/orders/create`);
- личные настройки (`/settings`);
- административная панель (`/admin`) для модерации контента и управления системой.

Доступ к данным разделам ограничивается middleware `requireAuth`, поэтому неавторизованный пользователь перенаправляется на страницу авторизации (рисунок 6).

**Рисунок 6 – Пример защищённого пользовательского раздела**

Для повышения безопасности при обработке форм в приложении используется CSRF-токен, а также серверные сессии с защищёнными cookie-параметрами (`httpOnly`, `sameSite`, `secure` в production). Это снижает риски подделки запросов и компрометации сессий.

Таким образом, реализация сайта Liniti покрывает основные сценарии платформы для креативных специалистов: презентация сервиса, публикация и просмотр работ, фильтрация контента, взаимодействие между пользователями и базовая административная модерация.
